## Problem Statement

El juego tiene 30 niveles pre-generados con soluciones únicas y verificadas. Durante testing manual, el desarrollador necesita poder ver rápidamente la solución de cualquier nivel para validar que las pistas son correctas y el tablero se presenta bien. No existe ninguna manera de ver las soluciones actualmente — hay que ir al JSON del puzzle y leerlo crudo.

## What Makes This Useful

- **Testing visual rápido**: ver el tablero completo resuelto en 2 clics vs. leer JSON crudo.
- **Validación de clues**: con los sospechosos colocados, las pistas se resaltan como "satisfechas" — se puede verificar que la lógica del CluePanel funciona correctamente.
- **Zero backend, zero state**: la solución ya está en el JSON del puzzle (campo `solution.placement`). Solo hay que renderizarla.

## Scope

**Feature:** Pantalla de soluciones accesible desde la pantalla principal.
**Audience:** Desarrollador durante testing manual (internal tool).
**Future:** Puede ocultarse en producción.

## Premises

1. **Los datos ya existen**: `puzzle.solution.placement` (Record<charId, CellId>) y `puzzle.solution.stolenObjectCellId` (CellId) están en cada `level-N.json`. No hay que cambiar el formato de datos.
2. **BoardGrid ya soporta read-only**: acepta `placement`, `glowingCellId` y `onCellClick`. Con `onCellClick = () => {}` el tablero es solo lectura.
3. **CluePanel puede reutilizarse directamente**: acepta `clues`, `board`, `characters`, `placement`. Con el placement de la solución, todas las pistas se mostrarán como satisfechas.
4. **La eliminación de filas/cols es derivada**: se puede calcular desde `solution.placement` con la misma lógica que GameScreen.

## Approach Selected

**Reutilizar componentes existentes (BoardGrid + CluePanel)** en dos pantallas nuevas: lista de niveles (SolutionsScreen) y detalle de solución (SolutionDetailScreen). Sin nuevos componentes visuales — solo wiring.

### Nuevas utilidades

#### `src/logic/boardUtils.ts` (nuevo)

```typescript
import type { Board, Character, CellId } from '../types'

export function computeEliminatedSets(
  board: Board,
  characters: Character[],
  placement: Partial<Record<string, CellId>>
): { eliminatedRows: Set<number>; eliminatedCols: Set<number> } {
  const eliminatedRows = new Set<number>()
  const eliminatedCols = new Set<number>()
  for (const char of characters) {
    const cellId = placement[char.id]
    if (!cellId) continue
    const cell = board.cells.flat().find(c => c.id === cellId)
    if (cell) {
      eliminatedRows.add(cell.row)
      eliminatedCols.add(cell.col)
    }
  }
  return { eliminatedRows, eliminatedCols }
}
```

Reemplaza el código inline de GameScreen.tsx:82-94 que se copia en los dos screens. GameScreen.tsx también se actualiza para usar esta función.

> **Nota:** `loadPuzzle` NO se extrae a una utilidad compartida. `import.meta.env.BASE_URL` causaría errores en Vitest si se importa en tests. Se define inline en cada screen (5 líneas idénticas — duplication acceptable).

## Success Criteria

1. Botón "Soluciones" en LevelSelect lleva a SolutionsScreen
2. Al hacer clic en un nivel: SolutionDetailScreen carga el puzzle JSON
3. Tablero muestra todos los sospechosos en sus posiciones correctas
4. La celda del objeto robado (`solution.stolenObjectCellId`) brilla en verde
5. Las filas/columnas eliminadas se muestran con X (consistente con GameScreen)
6. El CluePanel muestra todas las pistas resaltadas como satisfechas
7. Loading/error states funcionan igual que GameScreen
8. Botón "← Volver" en SolutionDetailScreen regresa a SolutionsScreen
9. `bun run build` pasa sin errores TypeScript
10. `bun run test` pasa sin cambios

| Codepath | Failure | Error handling |
|----------|---------|----------------|
| `loadPuzzle` en SolutionDetailScreen | HTTP error o JSON malformado | Mismo manejo que GameScreen: error state con "Reintentar" |

| `solution.stolenObjectCellId` no existe en el board | Bug en generator | No puede ocurrir: guaranteed by board invariant |
